# SEO basics

Search-oriented practices for content sites, marketing pages, and product landing pages.

## Technical foundation

- Crawlable HTML: meaningful `<title>`, one `<h1>`, logical heading order.
- Canonical URLs and clean paths (`/blog/post-slug` not `?id=123`).
- `robots.txt` and XML sitemap submitted in Search Console.
- HTTPS everywhere; fix mixed content and redirect chains.
- Core Web Vitals: LCP, INP, CLS (see [[Web Performance]]).

## On-page SEO

- Title tag: primary keyword + brand, ~50–60 characters.
- Meta description: compelling summary, ~150–160 characters (not a ranking factor but affects CTR).
- One clear H1 matching intent; H2/H3 for sections.
- Descriptive alt text on images; filenames like `team-photo.webp` not `IMG_0042.jpg`.
- Internal links with anchor text that describes the destination.

## Content intent

- Match search intent: informational (how-to), commercial (compare), transactional (buy/sign up).
- Answer the query in the first screen; expand with depth below.
- Update stale pages instead of only publishing new URLs.

## Structured data

- Use JSON-LD where it fits: `Organization`, `WebSite`, `Article`, `FAQPage`, `Product` (schema.org).
- Validate with Google Rich Results Test; avoid spammy or inaccurate markup.

## Local and international

- Local business: consistent NAP, Google Business Profile, local landing pages.
- `hreflang` for real multilingual sites; avoid machine-translated thin pages.

## Measurement

- Search Console: queries, coverage, enhancements.
- Analytics: organic landing pages, conversions by page, not vanity traffic alone.
